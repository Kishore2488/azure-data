# Deployment Workflows Guide\n\n## Overview\n\nThis guide provides deployment workflows for different Azure and Microsoft Fabric platforms.\n\n## Local Execution Workflow\n\n```\nClone Repository\n     ↓\nCreate Virtual Environment\n     ↓\nInstall Dependencies\n     ↓\nRun Application (python main.py)\n     ↓\nView Results (output/)\n     ↓\nDeactivate Environment\n```\n\n## Azure Databricks Workflow\n\n```\nCreate Resource Group\n     ↓\nCreate Databricks Workspace\n     ↓\nInstall & Configure Databricks CLI\n     ↓\nUpload Data to DBFS\n     ↓\nCreate Cluster\n     ↓\nCreate Notebook\n     ↓\nRun Notebook\n     ↓\nView Results\n```\n\n### Quick Start\n\n```bash\nbash scripts/deploy-databricks.sh\n```\n\n## Azure Synapse Analytics Workflow\n\n```\nCreate Resource Group\n     ↓\nCreate Storage Account\n     ↓\nCreate Synapse Workspace\n     ↓\nCreate Spark Pool\n     ↓\nUpload Data to Storage\n     ↓\nCreate Notebook\n     ↓\nRun Notebook\n     ↓\nView Results & Save Output\n```\n\n### Quick Start\n\n```bash\nbash scripts/deploy-synapse.sh\n```\n\n## Azure Container Instances Workflow\n\n```\nBuild Docker Image\n     ↓\nCreate Container Registry\n     ↓\nPush Image to Registry\n     ↓\nCreate Container Instance\n     ↓\nRun Container\n     ↓\nView Logs & Results\n```\n\n### Quick Start\n\n```bash\nbash scripts/deploy-container.sh\n```\n\n## Microsoft Fabric Workflow\n\n```\nCreate Workspace\n     ↓\nCreate Lakehouse\n     ↓\nCreate Notebook\n     ↓\nPaste & Run Spark Code\n     ↓\nTables Auto-Created as Delta\n     ↓\nCreate SQL Queries\n     ↓\nCreate Power BI Report\n     ↓\nSchedule Pipeline (optional)\n```\n\n### Manual Steps\n\n1. Go to https://app.powerbi.com/\n2. Create workspace → Create lakehouse\n3. Create notebook → Paste `fabric-notebook.py`\n4. Attach to Spark compute → Run cells\n5. Tables appear in Lakehouse\n6. Create SQL queries on tables\n7. Create Power BI visualizations\n\n## GitHub Actions Workflows\n\n### 1. Test Workflow (`.github/workflows/test.yml`)\n\nTriggers on:\n- Push to main\n- Pull requests\n\nRuns:\n- Install dependencies\n- Execute Spark application\n- Validate output files\n- Upload results as artifacts\n\n### 2. Databricks Deployment (`.github/workflows/deploy-databricks.yml`)\n\nTriggers on:\n- Push to main\n- Manual trigger\n\nSteps:\n- Configure Databricks CLI\n- Upload data and script\n- Create job\n- Run job\n\n### 3. Synapse Deployment (`.github/workflows/deploy-synapse.yml`)\n\nTriggers on:\n- Push to main\n- Manual trigger\n\nSteps:\n- Login to Azure\n- Upload data to storage\n- Trigger Synapse job\n\n### 4. Container Deployment (`.github/workflows/deploy-container.yml`)\n\nTriggers on:\n- Push to main\n- Manual trigger\n\nSteps:\n- Build Docker image\n- Push to ACR\n- Deploy to Container Instances\n- Monitor container\n\n## CI/CD Pipeline Flow\n\n```\nGitHub Push\n     ↓\n[Parallel Jobs]\n├─ Test Job\n│  ├─ Install dependencies\n│  ├─ Run Spark app locally\n│  └─ Validate outputs\n│\n└─ Deploy Jobs (if test passes)\n   ├─ Deploy to Databricks\n   ├─ Deploy to Synapse\n   └─ Deploy to Container Instances\n```\n\n## Environment Variables & Secrets\n\nRequired GitHub Secrets:\n\n### For Databricks\n- `DATABRICKS_HOST`: Workspace URL\n- `DATABRICKS_TOKEN`: Access token\n\n### For Synapse\n- `AZURE_CREDENTIALS`: Service Principal\n- `SYNAPSE_STORAGE_ACCOUNT`: Storage account name\n- `SYNAPSE_STORAGE_KEY`: Storage account key\n\n### For Container Registry\n- `AZURE_CREDENTIALS`: Service Principal\n- `AZURE_REGISTRY_NAME`: ACR name\n\n## Manual Deployment Commands\n\n### Databricks\n```bash\naz databricks workspace create --resource-group spark-sql-rg --name spark-sql-workspace --location eastus --sku premium\npip install databricks-cli\ndatabricks configure --token\ndatabricks fs cp data/ dbfs:/spark-sql-demo/ --recursive --overwrite\n```\n\n### Synapse\n```bash\naz synapse workspace create --name spark-sql-synapse --resource-group spark-sql-rg --storage-account synapsestorage123 --file-system synapsefs --sql-admin-login-user sqladmin --sql-admin-login-password P@ssw0rd123!\naz synapse spark pool create --name sparksql-pool --workspace-name spark-sql-synapse --resource-group spark-sql-rg --spark-version 3.1 --node-count 3\n```\n\n### Container Instances\n```bash\naz acr create --resource-group spark-sql-rg --name sparkcontainerreg --sku Basic\naz acr build --registry sparkcontainerreg --image spark-sql-demo:latest .\naz container create --resource-group spark-sql-rg --name spark-sql-container --image sparkcontainerreg.azurecr.io/spark-sql-demo:latest\n```\n\n## Monitoring & Logging\n\n### Databricks\n```bash\ndatabricks jobs list\ndatabricks runs list --job-id <job-id>\n```\n\n### Synapse\n```bash\naz synapse workspace show --name spark-sql-synapse --resource-group spark-sql-rg\n```\n\n### Container Instances\n```bash\naz container show --resource-group spark-sql-rg --name spark-sql-container\naz container logs --resource-group spark-sql-rg --name spark-sql-container\n```\n\n### Fabric\nGo to workspace → Monitor → View run history\n\n## Cleanup\n\n### All Azure Resources\n```bash\naz group delete --name spark-sql-rg --yes\n```\n\n### Specific Resources\n```bash\n# Databricks\naz databricks workspace delete --name spark-sql-workspace --resource-group spark-sql-rg\n\n# Synapse\naz synapse workspace delete --name spark-sql-synapse --resource-group spark-sql-rg\n\n# Container Registry\naz acr delete --name sparkcontainerreg --resource-group spark-sql-rg\n```\n\n## Troubleshooting\n\n### Common Issues\n\n1. **Permission Denied**\n   - Check Azure credentials\n   - Verify service principal has correct role\n\n2. **Docker Build Fails**\n   - Ensure Java is installed\n   - Check Dockerfile syntax\n\n3. **Container Won't Start**\n   - Check logs: `az container logs --resource-group spark-sql-rg --name spark-sql-container`\n   - Increase memory/CPU allocation\n\n4. **Databricks Job Fails**\n   - Check cluster status\n   - Verify data paths in DBFS\n   - Review job logs\n\n5. **Synapse Query Times Out**\n   - Increase pool size\n   - Optimize SQL query\n   - Check data sizes\n\n## Performance Tips\n\n1. **Databricks**: Use spot instances to reduce costs\n2. **Synapse**: Use dedicated SQL pools for frequent queries\n3. **Fabric**: Optimize query performance with appropriate table size\n4. **Containers**: Use auto-scaling for variable load\n\n## Cost Optimization\n\n- Databricks: $0.40-$1.50/DBU\n- Synapse: Pay-per-query model\n- Fabric: $5-$20/capacity/hour\n- Container Instances: $0.0015/second\n\nRecommendation: Use Fabric for integrated analytics, Synapse for data warehouse, Databricks for data engineering.\n
